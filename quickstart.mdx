# Quick Start

This guide will get you from zero to your first successful stash in under 5 minutes.

<Note>
Make sure you've [installed TokenStash](/installation) before starting this guide.
</Note>

## Your First Stash

<Steps>
  <Step title="Open your project in Claude Code">
    Navigate to any project directory where you want to use TokenStash. The project should be a git repository.

    ```bash
    cd /path/to/your/project
    ```

    Then start a Claude Code session in that directory.
  </Step>

  <Step title="Run your first scan">
    Let's build the initial stash by scanning your entire repository:

    ```bash
    /stash-scan
    ```

    TokenStash will:
    - Scan all files in your repository
    - Generate 3-5 line summaries for each file
    - Create an architecture overview
    - Build the initial context cache

    <Note>
    The first scan might take a minute depending on your project size. Subsequent updates are much faster.
    </Note>
  </Step>

  <Step title="Verify the stash was created">
    Check that the `.tokenstash/` directory was created:

    ```bash
    ls -la .tokenstash/
    ```

    You should see:
    ```
    .tokenstash/
    ├── architecture.md
    ├── decisions.md
    ├── rules.md
    └── files/
        └── [your file summaries]
    ```

    <Note>
    The `decisions.md` file will be empty at first. It gets populated when you run `/stash-save` after a working session.
    </Note>
  </Step>

  <Step title="Restart Claude and test">
    Close your current Claude Code session and start a new one in the same project directory.

    Your stash is now automatically injected into Claude's context. Try asking:

    ```
    "What does this codebase do?"
    ```

    Claude should be able to answer immediately without reading any files — because your stash is already in context.
  </Step>
</Steps>

## Understanding Your Stash

Let's look at what TokenStash created:

### architecture.md

Contains a high-level overview of your codebase:
- Project structure
- Key components and their relationships
- Architectural patterns
- Entry points and main workflows

### decisions.md

A log of decisions made across sessions. Initially empty, it gets populated when you run `/stash-save`.

### rules.md

Project-specific rules and conventions. Think of this as a more powerful `CLAUDE.md` file that persists and evolves.

### files/

Per-file summaries (3-5 lines each) for every file in your project. These help Claude quickly locate relevant code.

## Common Commands

Here are the commands you'll use most often:

### Save session summary

At the end of a working session:

```bash
/stash-save
```

This captures:
- What you decided and why
- Architecture changes
- Updated file summaries

### Force re-scan

If you want to rebuild the entire stash from scratch:

```bash
/stash-scan --force
```

<Warning>
The `--force` flag will overwrite all existing summaries. Use it when you've made major refactoring changes and want a fresh perspective.
</Warning>

### Clear the stash

To wipe the project stash completely:

```bash
/stash-clear
```

TokenStash will ask for confirmation before deleting anything.

## Tips for Better Stashes

<Note>
**Run `/stash-save` at the end of significant sessions**. The more you save, the better Claude's context becomes over time.
</Note>

- **Be consistent**: Save after implementing features, fixing bugs, or making architectural decisions
- **Use meaningful decisions**: When you run `/stash-save`, Claude will ask about decisions — give clear explanations
- **Scan after big changes**: Run `/stash-scan` after major refactoring or merging large branches
- **Commit your stash**: Share context with your team by committing `.tokenstash/` to git

## Troubleshooting

### Stash not loading

If Claude doesn't seem to have your stash context:
1. Check that `.tokenstash/` exists in your project root
2. Verify you started Claude Code in the correct directory
3. Try running `/stash-scan` again

### Summaries seem outdated

Run a force re-scan to rebuild everything:

```bash
/stash-scan --force
```

### Questions?

If you run into issues, check:
- That you're in a git repository
- That you have write permissions in your project directory
- That TokenStash is properly installed (try `/stash-scan --help`)
